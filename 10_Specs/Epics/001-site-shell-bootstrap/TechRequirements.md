# Site Shell Bootstrap — Technical Requirements

## Tech Stack

| Category | Choice | Version | Notes |
|---|---|---|---|
| Runtime | Bun | 1.3.11 | Standardized at monorepo root; all commands use `bun` |
| Language | TypeScript | ~6.x | Root devDependency; workspace gets its own tsconfig.json |
| Framework | Next.js | 16.x | App Router only; static export (`output: "export"`) |
| UI Library | React | 19.2.3 | Required by Next.js 16; includes React DOM |
| Testing (unit) | Vitest | latest stable | Not yet installed; new devDependency in workspace |
| Testing (e2e) | Playwright | ^1.x | `@playwright/test`; lives in separate `scriptor-web-test` workspace |

| CSS | Tailwind CSS v4 + PostCSS | ^4.x | `tailwindcss` + `@tailwindcss/postcss`; no separate config file |

## Testing Strategy

### Unit Tests

- **Location:** Co-located alongside source files — `20_Applications/scriptor-web/app/**/*.test.tsx` or `20_Applications/scriptor-web/app/**/*.test.ts`
- **Framework:** Vitest with jsdom environment
- **Naming:** `<ComponentName>.test.tsx` adjacent to the component file
- **Stub test requirement:** At least one passing test in `page.test.tsx` for AC-004. Minimum viable: verify the home page component renders without throwing (e.g., `render(<Page />)` with React Testing Library, assert non-null output).

### End-to-End Tests

- **Location:** `20_Applications/scriptor-web-test/tests/` — standalone workspace, separate from the site
- **Framework:** `@playwright/test`, Chromium only
- **Stub test requirement:** At least one passing test in `smoke.spec.ts` for AC-006. Minimum viable: navigate to `http://localhost:3000`, assert page title or a known element is present.
- **Server:** `bunx serve ../scriptor-web/out/ -p 3000` (serves the site's static build; requires `scriptor-web` build output to exist first)
- **WSL note:** In WSL environments, set `PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH` to the Windows Chrome executable when running headed.

## Constraints & Decisions

_Technical constraints from Research or the codebase._

- **Bun only** — no `npm`, `node`, `npx`. All installs and script runs use `bun` or `bunx`.
- **Biome only** — no ESLint, no Prettier. Tabs for indentation, double quotes in JS/TS.
- **Static export required** — `output: "export"` must be set in `next.config.ts`. No server-side rendering.
- **App Router only** — no `pages/` directory.
- **CSS Modules for custom components** — per existing web styling rules; no Tailwind utility classes in JSX.
- **shadcn/ui components** — live in `lib/components/ui/`; not modified after generation.
- **`.js` extensions on imports** — required for bundler-mode module resolution in TypeScript source files.
- **Import organization** — all imports at top of file, grouped: Bun built-ins → Node built-ins → external packages → internal. Enforced by Biome `organizeImports`.
- **Two new workspaces** — `20_Applications/scriptor-web` (the site) and `20_Applications/scriptor-web-test` (standalone E2E tests); both fit the existing `20_Applications/*` workspace glob.
- **Turbo output directory** — Next.js outputs to `out/` by default. `turbo.json` must include `out/**` in its `outputs` for the build task so Turbo caches build artifacts correctly.
- **CI/CD updates are in scope** — `working-directory` references in `ci.yml` and `release.yml` must be updated to `20_Applications/scriptor-web` as part of this epic's delivery. The defunct `tui-release` job in `release.yml` (lines 54–83) must also be removed.
