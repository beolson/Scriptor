**Description:**
Implement and test parsing of the `--repo <owner/repo>` CLI argument.

- Write failing tests for:
  - `--repo owner/repo` is parsed and returned as a string.
  - Missing `--repo` returns `null` (falls back to default).
  - Malformed values (e.g. missing slash, empty string) produce a descriptive error.

**Implementation Notes:**
- Created `source/src/cli/parseCli.test.ts` with 10 unit tests covering:
  - Happy path: `--repo owner/repo` returns `{ repo: "owner/repo" }`.
  - Realistic slug: `my-org/my-scripts` parsed correctly.

- Created `source/src/cli/parseCli.ts` exporting:

  - `parseCli(argv: string[]): CliArgs` — pure function, no I/O, no side effects.
  - Private `isValidRepo(value)` helper enforcing non-empty owner and repo on either side of exactly one slash.

## Task 3 — Config Service

**Status:** completed

**Description:**
Implement and test reading and writing the `~/.scriptor/config` YAML file.

- Write failing tests for:
  - Reading a valid config file returns a typed object (`{ repo?: string }`).
  - Reading a missing config file returns a default empty config (no error thrown).
  - Writing a config object creates or overwrites the file with correct YAML.
  - A corrupt/unparseable config file is handled gracefully (returns default config, does not crash).
- Implement `readConfig()` and `writeConfig(config)` in `source/src/config/`.
- Config path resolves relative to `os.homedir()` to support cross-platform testing with a temp directory override.

**Implementation Notes:**
- Created `source/src/config/config.test.ts` with 10 unit tests covering:
  - Happy path: valid YAML with `repo` key returns `{ repo: "owner/repo" }`.
  - File exists but has no `repo` key: returns `{}`.
  - File does not exist: returns `{}` (no error).
  - Corrupt/unparseable YAML: returns `{}` (no crash).
  - Non-object YAML (e.g. a list): returns `{}`.
  - Write creates the file with correct YAML content.
  - Write creates `.scriptor/` directory if it does not exist.
  - Write overwrites existing config file (old content gone).
  - Write of empty config (`{}`) produces `{}` YAML.
  - Round-trip: write then read returns the original object.
- Created `source/src/config/config.ts` exporting:
  - `ScriptorConfig` interface: `{ repo?: string }`.
  - `readConfig(baseDir?: string): Promise<ScriptorConfig>` — uses `Bun.file`, falls back to `{}` on missing/corrupt/non-object YAML.
  - `writeConfig(config: ScriptorConfig, baseDir?: string): Promise<void>` — uses `mkdirSync({ recursive: true })` + `Bun.write`, serializes with `js-yaml`.
  - Both functions default `baseDir` to `os.homedir()`.
- `bun test src/config/config.test.ts` → 10 pass, 0 fail.
- `bun run lint` → 0 errors.

## Task 12 — Script Execution Engine

**Status:** not started

**Description:**
Implement and test sequential child-process execution of scripts with log capture and failure halting.

- Write failing tests (using a mock child-process spawner) for:
  - `runScripts(scripts, logPath)` executes scripts sequentially in the resolved dependency order.
  - Each script's stdout/stderr is piped to the log file with banners/footers via the Log Service.
  - A non-zero exit code halts execution and returns a `ScriptFailureResult` describing which script failed.
  - Subsequent scripts in the queue are not started after a failure.
  - A successfully completed run returns a `ScriptSuccessResult` with the log file path.
  - Emits progress events (`pending`, `running`, `done`, `failed`) for each script so the TUI can react.
- No execution timeout — scripts run until they exit naturally.
- Implement `ScriptRunner` in `source/src/execution/`.
